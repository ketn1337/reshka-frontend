# Update Project Context Prompt

Изучи текущий проект `reshka-frontend` и актуализируй документацию для Claude Code.

## Цель

Поддерживать краткую и точную документацию, которая помогает Claude Code быстро понимать проект и не ломать архитектуру.

## Ограничения

- Не изменяй бизнес-логику.
- Не изменяй UI.
- Не изменяй API-клиенты.
- Не устанавливай зависимости.
- Не трогай `.env`, `.env.*`, секреты, токены и приватные файлы.
- Не изменяй `CLAUDE.md`, если пользователь не попросил явно.
- Не раздувай документацию.
- Не добавляй историю изменений, временные TODO и баг-репорты.
- Документация должна быть краткой и прикладной.

## Что нужно изучить

- `package.json`
- `src/App.tsx`
- `src/main.tsx`
- `src/api/`
- `src/components/`
- `src/admin/`
- `src/store/`
- `src/lib/`
- `src/booking.ts`
- `src/data.ts`
- `src/roomInventory.ts`
- новые или изменённые файлы, если они появились в проекте

## Какие документы обновлять

Обновляй только те документы, которые реально затронуты изменениями:

- `docs/PROJECT_CONTEXT.md` — если изменилась общая структура проекта или список документации.
- `docs/claude/ARCHITECTURE.md` — если изменилась архитектура, маршруты, структура приложения.
- `docs/claude/PUBLIC_APP.md` — если изменилась публичная часть.
- `docs/claude/ADMIN_APP.md` — если изменилась админка.
- `docs/claude/API.md` — если изменились API-клиенты, типы или контракты.
- `docs/claude/BOOKING_FLOW.md` — если изменилась логика бронирования.
- `docs/claude/UI_GUIDE.md` — если изменились UI-паттерны, Tailwind-стиль или работа с изображениями.
- `docs/claude/CHECKS.md` — если изменились scripts или команды проверки.

## Формат документации

- Markdown.
- Кратко.
- Без длинных туториалов.
- Без описания каждого файла подряд.
- Только то, что помогает будущему агенту работать с проектом.
- Если документ становится слишком большим — раздели его на более узкие документы и обнови ссылки в `docs/PROJECT_CONTEXT.md`.

## Финальный ответ

В конце напиши:

1. Что изучено.
2. Какие документы обновлены.
3. Почему обновлены именно они.
4. Какие проверки можно выполнить.
